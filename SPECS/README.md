# SPECS

This folder is the spec-driven source of truth for planning and task tracking.

Required structure per unit:
- `SPECS/units/<unit_name>/spec.md`
- `SPECS/units/<unit_name>/sdd.md`
- `SPECS/units/<unit_name>/task.md`

Global files:
- `SPECS/SDD.md`
- `SPECS/TASK.md`

Documentation governance follows `SPECS/SDD.md` (global rule) and applies to every unit.
